🧩 AI 增强功能
对AI回复或你自己发出的消息做文本清洗替换，比如过滤AI的口头禅、屏蔽词、统一格式。留空"替换为"则代表删除匹配到的内容。
开启后，除了最近几条消息，AI还会根据当前话题从更早的聊天记录里语义检索相关片段（需要你的API支持 /embeddings 接口）。同时也会用于角色的专属资料库检索。
默认优先使用上方的"副 API"拉取（没配置副API则用主API），拉取到的列表里选一个支持向量/embedding的模型即可。
部分角色卡自带的HTML卡片（比如状态栏的展开/收起按钮）依赖内嵌的 <script> 才能工作，浏览器默认不会执行这类动态插入的脚本，开启后才能生效。但这意味着聊天内容里只要出现脚本就会真的运行，存在安全风险——只有在明确信任自己导入的角色卡来源时才建议开启，不确定的话保持关闭。
功能越加越多，每次请求塞给AI的内容也越滚越大。这里可以限制各部分最多占用多少字数——世界书会按优先级/权重自动只保留预算内的高优先条目，其余悄悄跳过，不影响核心设定。

🔌 插件
插件是可以导入/导出、分享给别人使用的"打包好的功能"，不用每次都改代码。支持四种类型：提示词规则（往AI脑子里塞一条固定指令）、快捷动作（聊天框上方加一个按钮，点一下发送预设指令，比如掷骰子）、宏（扩展 {{xxx}} 占位符）、脚本钩子（进阶，自定义JS，请只导入信任来源的脚本）。

新建插件

⚠️ 进阶功能：这段代码会被直接执行。只导入你信任来源的脚本！
函数签名：function(char, context, applyMacros) { return "要注入的文本"; }
函数签名：function(char, sessionId, response, applyMacros) { ... }，response是AI返回的JSON对象，可以直接读取/修改它的字段（比如 response.affinityDelta、response.replies），全局变量（myCharacters、saveAllData、showToast等）都能直接用。
函数签名：function(applyMacros) { ... }，这段代码会在每次打开网页、数据读取完毕后自动跑一次，document/window/全部页面里的函数和变量都能直接用，可以用来注入自定义的导航按钮、页面、弹窗、样式等。
